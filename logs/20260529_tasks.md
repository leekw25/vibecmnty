# 2026-05-29 - 관리자 통합 대시보드 구현

## 목표
- 관리자만 접근 가능한 통합 대시보드 및 기능별 관리 페이지 구현
- `/admin-dashboard` 경로에 총 회원/게시글/투표·설문/일정 통계와 최근 활동 피드 구성
- 공지사항/게시글/일정/투표/설문/사용자 관리 및 시스템 관리 화면 구현
- 403(권한) 처리, 로그인 필수, Django Admin 연동 링크 포함
- 업무 보고(`tasks.md`) 및 작업일지(`logs.md`) 작성

## 구현 내용
- `admin_dashboard/views.py`
  - 관리자 인증 공용 데코레이터 `_require_admin` 추가 (`login_required` + 관리자 판정 + PermissionDenied)
  - 메인 대시보드 통계 계산, 최근 활동 피드 조합
  - 공지사항 관리, 게시글/댓글 관리, 일정/투표/설문 관리, 사용자 관리, 시스템 관리 뷰 구현
  - 사용자 역할 변경/활성/비활성 처리 엔드포인트 추가
- `admin_dashboard/urls.py`
  - `/admin-dashboard/` 아래 관리 화면 라우팅 추가 (`notices`, `posts`, `events`, `polls`, `surveys`, `users`, `system`, `user_action`)
- 템플릿 구성 (`templates/admin_dashboard/`)
  - 공통 레이아웃: `base.html` (사이드바 + 헤더)
  - 메인 대시보드: `home.html`
  - 기능별 관리 화면:
    - `notice_management.html`
    - `post_management.html`
    - `event_management.html`
    - `poll_management.html`
    - `survey_management.html`
    - `user_management.html`
    - `system_management.html`

## 완료 상태
- 구현 완료(코드 반영)
- 권한 제어(로그인 + 관리자 403) 적용
- Django Admin 진입 링크 제공

## 비고
- 사용자 활동 통계/레포트용 항목은 현재 모델 구조 기반으로 구현했고, 신고 기능 전용 필드는 아직 없어 신고 큐 UI는 추후 확장 포인트로 처리.
