# PRD (Product Requirements Document)

## 1. 프로젝트 개요
**프로젝트명:** Vibe Community  
**목적:** 공지사항, 소통, 일정, 투표, 회원/권한, 관리자 기능을 갖춘 소규모 커뮤니티 웹서비스를 빠르게 구축하고 운영 가능한 구조로 만든다.  
**핵심 가치:** 보안성, 유지보수성, 바이브코딩 생산성, 빠른 기능 확장.

## 2. 문제 정의
- 커뮤니티 운영자가 공지·이벤트·일정을 한 곳에서 관리할 도구가 필요함
- 사용자 참여(댓글/좋아요/조회수/이미지 공유)가 쉬워야 함
- 운영자가 기본 통계를 보고 운영할 수 있어야 함
- 개발 인력이 거의 없더라도 향후 기능 확장이 쉬워야 함

## 3. 목표
1. 3~4개월 내 핵심 MVP를 안정적으로 릴리즈
2. SQLite 기반으로 시작해 초기 운영 안정성 확보
3. 권한 분리(관리자/일반 회원/비로그인)로 악성 사용률 방지
4. 추후 PostgreSQL 전환 또는 클라우드 호스팅 확장 가능
5. AI 협업 기반 개발이 가능한 코드 구조 확보

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 공지사항
- 관리자만 공지 작성/수정/삭제 가능
- 회원/비회원 모두 조회 가능
- 중요 고정/일반 공지 구분
- 공지 이미지 첨부 지원(초기 1회 업로드 제한 허용)

### 4.2 소통 공간
- 자유게시판 CRUD
- 게시글 조회수 집계 (중복 조회 방지 정책 필요)
- 좋아요 토글(1인 1회)
- 댓글/대댓글 계층 구조(최소 2단)
- 게시글·댓글 이미지 첨부
- 익명 악의성 제어(신고/차단 정책)

### 4.3 주요 일정 (캘린더)
- 일정 등록/수정/삭제(관리자 중심)
- 카테고리 구분(행사/점검/기타)
- 월간/주간 뷰
- 마감 임박 알림(추가 구현: 알림 채널 분리 가능)

### 4.4 투표(설문 확장)
- 단일/복수 선택 투표
- 익명 투표 옵션
- 중복 참여 제한(로그인 기반)
- 투표 결과 집계 및 시각화
- 마감일, 비공개 결과 토글

### 4.5 회원/인증
- 회원가입(이메일, 비밀번호)
- 로그인/로그아웃
- 세션 기반 인증 및 만료 정책
- 프로필 조회/수정
- 권한(Role): `USER`, `ADMIN`, 필요시 `MODERATOR`

### 4.6 관리자 대시보드
- 사용자/게시글/댓글/신고/일정/투표 관리
- 활동량 통계(주간/월간)
- 상태 제어(정지, 블라인드, 삭제)

## 5. 비기능 요구사항 (Non-Functional Requirements)
- 보안: 비밀번호 해시, 세션/CSRF 보호, 입력 값 검증, 권한 검증 필수
- 확장성: 신규 기능(채팅, 알림, 포인트) 추가가 쉬워야 함
- 성능: 동시 접속 초기가능 수준 기준에서 기본 응답 500ms 이하 목표(단순 조회)
- 유지보수성: 도메인 단위 구조 + 테스트 가능한 서비스 계층 분리
- 사용성: 직관적 UI, 모바일 우선, 핵심 동작은 3클릭 이내 완료

## 6. 아웃 오브 스코프 (초기)
- 실시간 채팅
- 실시간 푸시 알림(웹/모바일 push)
- 결제/구독 모델
- 다국어 지원(후속)

## 7. 성공 지표 (KPI)
- 일간 게시글 작성 수 ≥ 20건/커뮤니티(런칭 2개월차 목표)
- 댓글 반응률 ≥ 게시글 대비 30%
- 공지 조회 이탈률 저감(목표: 공지 목록 클릭 후 상세 조회율 50%↑)
- 관리자 승인/대응 작업 시간 평균 30% 감축

## 8. 위험요소
- 상태관리 복잡화로 기능 추가 속도 저하
- 권한 실수로 인한 노출/탈취 위험
- 과도한 비동기 처리로 데이터 정합성 이슈(조회수/좋아요)
- SQLite 동시성 한계(사용량 증가시 PostgreSQL 전환 필요)

## 9. 향후 로드맵
MVP(공지·게시판·회원·일정) → 투표·통계 강화 → 신고/제재 자동화 → 알림 시스템 확장.
