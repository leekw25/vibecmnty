# DP (Design Principles)

## 1. UI/UX 철학
- **사용 목적 중심:** 게시판/공지/일정/투표는 장식보다 동작이 우선
- **인간 중심 정보 구조:** 목록에서 세부로 진입이 자연스러워야 하며, 사용자 흐름은 3단계 이하로 단순화
- **응답성:** 모바일 우선(최소 360px), PC에서 가독성 유지
- **일관성:** 동일 동작(버튼/링크/상태)은 동일한 시각 언어 사용
- **접근성 기본 준수:** 키보드 포커스, 라벨, 대비, ARIA 기본 속성

## 2. 정보 아키텍처
### 공통 탐색
- 최상단: 공지 / 소통 / 일정 / 투표 / 관리자
- 각 섹션에서 상세 동작(작성/조회/댓글/투표/관리)으로 내려가는 1-depth 구조

### 콘텐츠 규칙
- 공지는 중요도 순으로 정렬(중요 고정 + 최신 우선)
- 게시판은 “최신/인기” 탭 제공
- 일정은 월간 뷰 + 당일 하이라이트
- 투표는 결과 공개/비공개를 한눈에 확인 가능

## 3. 인터랙션 원칙
- 새 글/댓글은 즉시 반영되되, 실패 시 명확한 복구 문구 제공
- 삭제/중요 상태 변경은 확인 모달 필수
- 좋아요·투표는 낙관적 업데이트 사용 여부를 로그로 추적하고, 실패 시 롤백 처리

## 4. 제약사항
- 과도한 애니메이션은 배제(성능 우선)
- 관리자 페이지는 작업 효율성이 우선되어야 하므로 한 화면에서 다건 조작이 가능해야 함
- 이미지 업로드는 용량/형식 제한(초기: JPG/PNG/WebP, 5MB 이하)
- 긴 텍스트는 줄 바꿈·최대 높이 제한으로 가독성 유지

## 5. 디자인 토큰 가이드(초안)
- 색상: 배경/텍스트 대비 최우선(접근성 WCAG AA 기준 지향)
- radius: 카드/입력요소는 8px 이하
- 간격: 4/8/12/16 기반 스케일
- 타이포: 한글 가독성 우선, 영문/숫자 보강 폰트 동시 적용

## 6. 비기능 UX 제약
- 로딩 스피너보다 의미있는 빈 상태 화면 사용
- 비로그인 사용자는 글쓰기/좋아요/투표 동작에서 로그인 유도
- 404/500 오류 페이지는 검색 가능성 있는 회귀 경로(목록/홈)로 즉시 전환 링크 제공
